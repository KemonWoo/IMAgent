# IMAgent 交付方案 V2.0

## 核心原则

**AI Native：** Agent 全权部署，人类只需一句话 + 选菜单。无代码用户是首要服务对象。

**零服务器成本：** 内网穿透方案全覆盖，人类不需要拥有公网服务器。

**仓库决定路径：** Gitee → 国内穿透菜单，GitHub → 海外穿透菜单。不依赖 IP 检测。

## 双仓库/双分支策略

**方案 A：双仓库**
- GitHub `KemonWoo/IMAgent` — 海外版 AGENTS.md + README
- Gitee `KemonWoo/IMAgent` — 国内版 AGENTS.md + README
- 源码相同，仅 AGENTS.md / README.md / 穿透菜单不同
- 维护成本：需手动同步源码改动到两边

**方案 B：双分支**
- `main` 分支 → GitHub 海外版
- `gitee` 分支 → Gitee 国内版
- 差异文件：AGENTS.md, README.md, deploy.sh
- 源码同步：`git merge main` 到 gitee 分支即可

**方案 C：单源码 + 生成脚本（推荐）**
- 一个源码库，不分叉
- AGENTS.md 使用模板变量
- `scripts/generate-docs.sh` 根据 `TARGET=gitee|github` 生成对应版本
- Gitee/GitHub 各部署生成后的文档
- 维护成本最低，源码单一

## 人工干预最小化策略

| 场景 | 干预点 | 如何最小化 |
|------|--------|----------|
| Linux 本机 | sudo 危险指令 | Hermes Approve Always |
| Windows 本机 | UAC 弹窗 | 避开需要管理员权限的操作 |
| 远程服务器 | SSH 审批 | Approve Always 或预设密钥 |
| 穿透注册 | 手机号/验证码 | 选方案 1（全自动无需注册） |
